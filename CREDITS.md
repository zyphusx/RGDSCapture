## Open Source Libraries

RGDSCapture would not exist without the following projects and the people behind them.

### FFmpeg
The industry-standard multimedia framework powering H.264 decoding and MP4 recording in this application.
Maintained by a large community of contributors worldwide.
https://ffmpeg.org

### FFmpeg.AutoGen — Ruslan Balanukhin
Auto-generated .NET bindings that make FFmpeg accessible from C# without hand-written P/Invoke code.
https://github.com/Ruslan-B/FFmpeg.AutoGen

### SSH.NET — The SSH.NET Contributors
A mature, well-maintained SSH client library for .NET. Handles everything from key exchange to command execution cleanly and reliably.
https://github.com/sshnet/SSH.NET

### NAudio — Mark Heath
A comprehensive audio library for .NET that made building low-latency Line-In monitoring significantly simpler than working with the Windows audio APIs directly.
https://github.com/naudio/NAudio

### BouncyCastle — The Legion of the Bouncy Castle
The cryptographic foundation that SSH.NET relies on for secure connections.
https://www.bouncycastle.org

## Special Thanks

A particular thanks to the FFmpeg project — without a freely available, high-quality H.264 decoder there would be no practical path to building a software capture card like this in a weekend.

And to the broader open-source .NET ecosystem: the fact that a Windows desktop application with real-time video decoding, SSH control, and audio monitoring can be built with freely available, well-documented libraries is genuinely remarkable.
